# Administrative tasks in System Center 2016 - Service Manager

The articles in this section provide concepts that are important to Service Manager administrators, and it includes procedures that Service Manager administrators must perform to configure Service Manager. These procedures are typically performed one time after Service Manager is deployed. The procedures in this guide help you configure Service Manager to match the policies and requirements defined by your organization. These articles provide information and detailed procedures to:

- Create, import, and export management packs.
- Import data into Service Manager by using connectors.
- Manage configuration items.
- Set configuration settings that streamline the creation of incidents.
- Configure service-level management.
- Create and use workflows.
- Streamline change and activity management.
- Configure settings and workflows for release management.
- Configure Service Manager to create incidents based on desired configuration management reports from System Center Configuration Manager.
- Configure notifications.
- Use the service catalog.
- Use groups to manage configuration items, use queues to manage work items, and use lists to customize forms.
- Use runbooks in Service Manager and System Center - Orchestrator to automate procedures.
- Use unsealed management packs to edit the properties of views, list items, or templates.
- Use user roles and to define a scope of activities that can be performed.
- Change the Operational System and Workflow Run As accounts.
- Create and search for knowledge articles.
- Configure and use Service Manager cmdlets to automate tasks.
- View, schedule, and troubleshoot extract, transform, and load (ETL) jobs in the Service Manager data warehouse.
- Use tasks to troubleshoot computer problems.
